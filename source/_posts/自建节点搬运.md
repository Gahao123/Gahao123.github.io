## 第一步,租用云端服务器VPS
这个是VPN的主机,搭VPN的方法到处都一样,但能不能好用还取决于云厂商

推荐选择美国西海岸(洛杉矶/西雅图),新加坡,日本大阪(比东京干净);尽量避开Vultr的日本东京节点,是重灾区,开出来很多新IP就被封;如果第一次开出来ping不通,在VPS后台点"重建"换个IP即可,通常不额外收费

### 确定服务商之后
在服务商后台创新服务器(Droplet/Instance),选配置如下:

|配置项|说明|
|---|---|
|Shared CPU|5`$`/m那一个,Automatic Backups自动备份弄成Disabled因为要收费,必须是Instance with Public IP,地址IPv4和IPv6都选上(虽然选了IPv6,但是使用还是用IPv4,IPv6容易出各种问题,当个备用就行,反正IPv6地址没额外成本)|
|Filter Locations|选服务器地址,按上面的tip选一个|
|Image Selection and Configuration|Operating System|
|System Image|Ubuntu 22.04LTS或24.04这俩稳定|
|Server Size|5`$`/mo(1 CPU, 1GB RAM)反正就是最便宜那个|

- 创完开机后在这个服务器的控制台就有IP和root密码
- 看流量在右上角Org那里的Bandwidth Usage就能看到,最上面的表上方Pool是总量,下一行Usage是已用量,这个是每月的限额,基本是随便用
- IPv4改IPv6订阅地址:直接在订阅的IPv4地址的地方换成IPv6地址,然后外面加上中括号`[]`即可,建议手机使用IPv6的节点(似乎更快更稳定?)

## 第二步,SSH连服务器
- 打开cmd:`ssh root@你的服务器IP` ; 如果有东西,yes,还有输入密码,就是服务器的root摩玛 -> ssh是足够安全的,放心
- 成功连接后,会有欢迎界面,还有`root@xxx:~#`提示符

## 第三步,安装X-ui面板,完成服务端节点创建
1. 服务器的`root@xxx:~#`下输入`bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)`,(这是目前Github最受欢迎的3x-ui分支,看源码就去Github搜3x-ui) -> 这个的原版x-ui因为更新慢,所以不那么推荐了
会提示基础设置:设置访问端口y,然后输入一个10000-65000之间的数字,然后会弹出设置证书,这里防火墙端口80还未开放一般无法成功
2. 【**防火墙**】:首先是VPS上执行`ufw status`,看得有`443 (v6) ALLOW`和**你设定端口**的`ALLOW`,若没有,那就`ufw allow 443`和`ufw allow 你设置的端口号`,然后`ufw reload`(在线端口同样的),如果想弄https安全连接,要临时IP证书的话还得`ufw allow 80`,这三个都开放后服务器输入`x-ui`,找`19-SSL证书`里面那个`6天auto-renew`的,不要写IPv6地址,只生成IPv4证书,控制面板只用IPv4登即可,不影响IPv4和IPv6订阅,这里生成完证书后就会在绿字里出来一个地址`https://服务器IP:设置的端口/一串码`,这个是服务器VPN的控制台,然后建议再弄一下`6-重置用户名/密码`,把`https`的用户名密码弄成自己记得住的
3. 【这里一般采用这个顺序】因为`https`是安全的,这样一上来就`https`可以避免由于`http`不安全而带来的风险
4. 然后打开浏览器,进入上面出来那个地址`https://服务器IP:设置的端口/一串码`(**把证书弄了之后在绿字里就会出来一个https的网址,直接上这个就行**),就进入了服务器VPN的控制台,输入刚才`7-重置用户名/密码`弄的账号密码 ; 看地址也可以直接`x-ui settings`就能看到 ; 如果账号密码忘了只能在服务器那重置,就是x-ui的`7-Reset账号密码` ; 如果网页打不开,通常是因为服务器的云防火墙(Firewall/Security Group)未开放该端口,去VPS后台防火墙设置添加入站规则放行该端口
5. 进去之后左边面板设置->订阅设置里有个"URL路径"值默认是`/sub/`,改一下在后面加上一个乱码,网上搜一个随机密码生成器粘进去就行
6. 然后创建Reality节点,左侧面板设置Inbounds的添加入站Add Inbound,填备注Remark随便填一个,协议Protocol选VLESS,端口Port填443,第四项安全选Reality,uTLS选chrome,目标网站Dest填`www.microsoft.com:443`(要伪装的对象),Server Names(SNI)写`www.microsoft.com`,公钥和私钥Keys点下边的获取新证书Get New Cert自动生成一对密钥 -> 检查一下后点创建Add
7. 然后创建客户端,左侧选"客户端",然后添加客户端,只需要下面的"选择一个关联入站"选上一步创的那个入站,然后创建即可,这样节点就创建完毕了
8. 【**获得订阅链接**】找到刚创的这个客户端,点左边第二个客户端信息,在下面"复制链接"那一项里就可以得到`vless://`开头的节点(每一个客户端给一个单独的人用,可以监测) -> 也可以直接扫码

## 第四步,客户端配置
- Reality协议较新,得用最新版v2,下载就在github的仓库,去搜就行,作者是`2dust`(N和NG都是这个作者),**注意不是下载Code**,而是右边`About`那一列下边一点的`Releases`,这个才是下载的地方,选一个最新版的下即可,一般是`v2rayN-windows-64.zip`或`v2rayNG_x.x.x_arm64-v8a.apk`
- Windows:v2rayN,复制刚才的`vless://`直接Ctrl+V导入即可
- Android:v2rayNG,右上角+号选从剪切板导入,或扫描二维码

## 常见问题
- 验证VPN是否生效:访问`https://ip.sb/`,若显示服务器IP,则Reality已生效 ; 测试IPv6情况使用`https://test-ipv6.com`
- 若客户端报错certificate has expired,可能是系统时间不对,Reality对时间检测严格,确保时间误差不超过1min
- 【注意】Vultr上关机也会继续收费,因为磁盘+IP还在占资源,想不收费就只能销毁实例(东西也都没有了)
- 如果*能ping通+控制面板能打开+配置也查不出问题*但是Ipv4和Ipv6都连不上,那就是端口搞坏了,换端口433为8443/2053/2083/2087/2096或10000–60000任意一个5位数 -> 记得删除原来的端口防火墙`ufw delete allow 443`然后放行新的端口`ufw allow 你设置的端口号`
- 【*关于地址*】凡是要填地址的地方都去浏览器搜"美国地址生成器免税州"

## 检测有没有被墙
在站长之家的在线ping工具(ping.chinaz.com)或者ITDOG.cn，输入你的服务器IP地址，选择TCP测试
如果国外节点全是绿色(成功)，而国内节点全是红色(超时)，那么99%的概率是IP被墙了，需要换IP ; IPv6要是也挂了就先去确认一下自己连的网**有IPv6地址**,重新DHCP一下,**自己连的网没有IPv6那必用不了IPv6**(手机流量一般是可以用IPv6订阅的)
【先创建新服务器，等新IP分配好之后，再删旧服务器】顺序反了的话，系统可能把同一个IP再次分配给你，就白忙活了，新服务器IP出来之后，用ping工具快速测一下，能ping通再用
