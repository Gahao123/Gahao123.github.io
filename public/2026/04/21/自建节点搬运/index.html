自建节点搬运

写在前面

搬运的狠r这个
还有Reality的这个
用的是VLESS+Reality，安全级别最高，且有可视化流量管理后台
资源有限,一个省级节点工程师也就几十人这个量级
注意便宜VPS的线路!=稳定网络
只要没有在设备上安装过乱七八糟的根证书,并且在HTTPS下浏览网页,那运营商最多只能看到你从哪来,到哪去,但是看不到你做了什么 -> 检查详见TLS那里的最后

必要清单

  1. zfb，可Vultr支付/BandwagongHost，若原生不支持zbf可宝搜索礼品卡/充值代充值，直接帮充到账户，也可以成品号
  2. 一个电子邮箱(Gmail/Outlook/QQ邮箱均可)，若没有宝搜索Gmail成品号，约5￥-15￥，拿到后第一时间改密码绑手机号二次验证(但成品号有被原主找回风险)
  3. 电脑，Win10+
  4. 30分钟空闲时间

第一步,租用云端服务器VPS

这个是VPN的主机,搭VPN的方法到处都一样,但能不能好用还取决于云厂商

服务商,Invoice账单、发票 , Free Tire Program免费套餐计划

厂家 说明
Vultr 这个++,界面最简单,支持zfb,但东j/热门地区容易被封(密码+25并且+大写,然后查看账单在右上角那个Org里,不是用户那里)
下面备选 万一Vultr寄了
DigitalOcean 6$/m起
Linode(Akamai) 5$/m起
BandwagongHost zg用户扎堆,支持zfb,价格稍高,需注意选冷门机房
Oracle Cloud甲骨文 Always Free方案送两台ARM小机,永久免费,但申请要求高需真实信用卡,新号容易被收回
RackNerd/Hostinger 月付2-3$的VPS,稳定性和IP品质次一些但便宜

推荐选择美国西海岸(洛杉矶/西雅图),新加坡,日本大阪(比东京干净);尽量避开Vultr的日本东京节点,是重灾区,开出来很多新IP就被封;如果第一次开出来ping不通,在VPS后台点"重建"换个IP即可,通常不额外收费

确定服务商之后

在服务商后台创新服务器(Droplet/Instance),选配置如下:

配置项 说明
Shared CPU 5$/m那一个,Automatic Backups自动备份弄成Disabled因为要收费,必须是Instance with Public IP,地址IPv4和IPv6都选上(虽然选了IPv6,但是使用还是用IPv4,IPv6容易出各种问题,当个备用就行,反正IPv6地址没额外成本)
Filter Locations 选服务器地址,按上面的tip选一个
Image Selection and Configuration Operating System
System Image Ubuntu 22.04LTS或24.04这俩稳定
Server Size 5$/mo(1 CPU, 1GB RAM)反正就是最便宜那个
  • 创完开机后在这个服务器的控制台就有IP和root密码
  • 看流量在右上角Org那里的Bandwidth Usage就能看到,最上面的表上方Pool是总量,下一行Usage是已用量,这个是每月的限额,基本是随便用
  • IPv4改IPv6订阅地址:直接在订阅的IPv4地址的地方换成IPv6地址,然后外面加上中括号[]即可,建议手机使用IPv6的节点(似乎更快更稳定?)

第二步,SSH连服务器

  • 打开cmd:ssh root@你的服务器IP ; 如果有东西,yes,还有输入密码,就是服务器的root摩玛 -> ssh是足够安全的,放心
  • 成功连接后,会有欢迎界面,还有root@xxx:~#提示符

第三步,安装X-ui面板,完成服务端节点创建

  1. 服务器的root@xxx:~#下输入bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh),(这是目前Github最受欢迎的3x-ui分支,看源码就去Github搜3x-ui) -> 这个的原版x-ui因为更新慢,所以不那么推荐了
    会提示基础设置:设置访问端口y,然后输入一个10000-65000之间的数字,然后会弹出设置证书,这里防火墙端口80还未开放一般无法成功
  2. 【防火墙】:首先是VPS上执行ufw status,看得有443 (v6) ALLOW和你设定端口的ALLOW,若没有,那就ufw allow 443和ufw allow 你设置的端口号,然后ufw reload(在线端口同样的),如果想弄https安全连接,要临时IP证书的话还得ufw allow 80,这三个都开放后服务器输入x-ui,找19-SSL证书里面那个6天auto-renew的,不要写IPv6地址,只生成IPv4证书,控制面板只用IPv4登即可,不影响IPv4和IPv6订阅,这里生成完证书后就会在绿字里出来一个地址https://服务器IP:设置的端口/一串码,这个是服务器VPN的控制台,然后建议再弄一下6-重置用户名/密码,把https的用户名密码弄成自己记得住的
  3. 【这里一般采用这个顺序】因为https是安全的,这样一上来就https可以避免由于http不安全而带来的风险
  4. 然后打开浏览器,进入上面出来那个地址https://服务器IP:设置的端口/一串码(把证书弄了之后在绿字里就会出来一个https的网址,直接上这个就行),就进入了服务器VPN的控制台,输入刚才7-重置用户名/密码弄的账号密码 ; 看地址也可以直接x-ui settings就能看到 ; 如果账号密码忘了只能在服务器那重置,就是x-ui的7-Reset账号密码 ; 如果网页打不开,通常是因为服务器的云防火墙(Firewall/Security Group)未开放该端口,去VPS后台防火墙设置添加入站规则放行该端口
  5. 进去之后左边面板设置->订阅设置里有个"URL路径"值默认是/sub/,改一下在后面加上一个乱码,网上搜一个随机密码生成器粘进去就行
  6. 然后创建Reality节点,左侧面板设置Inbounds的添加入站Add Inbound,填备注Remark随便填一个,协议Protocol选VLESS,端口Port填443,第四项安全选Reality,uTLS选chrome,目标网站Dest填www.microsoft.com:443(要伪装的对象),Server Names(SNI)写www.microsoft.com,公钥和私钥Keys点下边的获取新证书Get New Cert自动生成一对密钥 -> 检查一下后点创建Add
  7. 然后创建客户端,左侧选"客户端",然后添加客户端,只需要下面的"选择一个关联入站"选上一步创的那个入站,然后创建即可,这样节点就创建完毕了
  8. 【获得订阅链接】找到刚创的这个客户端,点左边第二个客户端信息,在下面"复制链接"那一项里就可以得到vless://开头的节点(每一个客户端给一个单独的人用,可以监测) -> 也可以直接扫码

第四步,客户端配置

  • Reality协议较新,得用最新版v2,下载就在github的仓库,去搜就行,作者是2dust(N和NG都是这个作者),注意不是下载Code,而是右边About那一列下边一点的Releases,这个才是下载的地方,选一个最新版的下即可,一般是v2rayN-windows-64.zip或v2rayNG_x.x.x_arm64-v8a.apk
  • Windows:v2rayN,复制刚才的vless://直接Ctrl+V导入即可
  • Android:v2rayNG,右上角+号选从剪切板导入,或扫描二维码

常见问题

  • 验证VPN是否生效:访问https://ip.sb/,若显示服务器IP,则Reality已生效 ; 测试IPv6情况使用https://test-ipv6.com
  • 若客户端报错certificate has expired,可能是系统时间不对,Reality对时间检测严格,确保时间误差不超过1min
  • 【注意】Vultr上关机也会继续收费,因为磁盘+IP还在占资源,想不收费就只能销毁实例(东西也都没有了)
  • 如果能ping通+控制面板能打开+配置也查不出问题但是Ipv4和Ipv6都连不上,那就是端口搞坏了,换端口433为8443/2053/2083/2087/2096或10000–60000任意一个5位数 -> 记得删除原来的端口防火墙ufw delete allow 443然后放行新的端口ufw allow 你设置的端口号
  • 【关于地址】凡是要填地址的地方都去浏览器搜"美国地址生成器免税州"

检测有没有被墙

在站长之家的在线ping工具(ping.chinaz.com)或者ITDOG.cn，输入你的服务器IP地址，选择TCP测试
如果国外节点全是绿色(成功)，而国内节点全是红色(超时)，那么99%的概率是IP被墙了，需要换IP ; IPv6要是也挂了就先去确认一下自己连的网有IPv6地址,重新DHCP一下,自己连的网没有IPv6那必用不了IPv6(手机流量一般是可以用IPv6订阅的)
【先创建新服务器，等新IP分配好之后，再删旧服务器】顺序反了的话，系统可能把同一个IP再次分配给你，就白忙活了，新服务器IP出来之后，用ping工具快速测一下，能ping通再用

额外,可加上密码管理部署

找那些开源密码管理器,比如Bitwarden这种,自己部署,密文数据完全在你自己的服务器上
但是注意,这样的话服务器的安全维护(系统更新、Docker 镜像更新、防火墙配置)得自己搞,出了漏洞自己扛风险 ; 必须要HTTPS ; 东西还要自己备份 -> 自建密码管理,那就得信任自己的运维能力才行

记一次Vultr安全更新强制重启

7.22收到邮件,Vultr发现了一个影响Linux KVM虚拟化环境的严重漏洞,需要对基础设施进行安全补丁更新,因此要求对受影响VM进行一次强制重启,会在2026-07-27 07:00 UTC进行,换成UTC+8就是2026-07-27 15:00
说明Vultr还算正规,一些小厂可能遇到这种KVM级漏洞会不通知、拖几个月或不维护宿主机,大型云厂商一般会强制维护
重启后看是否正常运行systemctl status x-ui,正常输出enabled
