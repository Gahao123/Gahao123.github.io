Java正则特殊

在 Java 中正则表达式中则需要有两个反斜杠才能被解析为其他语言中的转义作用，在 Java 的正则表达式中，两个(反斜杠)代表其他语言中的一个(反斜杠)，表示一位数字的正则表达式是(两个反斜杠)d，而表示一个普通的反斜杠是(两个反斜杠)
一些正则语法:

  • ^匹配开始,$匹配结尾
  • 跟在要匹配的字符后面:*匹配0~多次,+匹配1~多次,?匹配0~1次,{n}正好匹配n次,{n,}匹配n~多次,{n,m}匹配n~m次,.匹配单个任意字符
  • x|y匹配x或y(z|food匹配z或food,而(z|f)ood匹配zood或food)
  • [xyz]匹配包含的一个字符,[^xyz]匹配不包含的一个字符,[a-z]匹配指定范围内一个,[^a-z]匹配不在范围内的一个
  • \d数字字符0-9,\D非数字(不是0-9),\n换行,\s空白字符,\S非空白字符,\w字类字符大小写字母+数字+下划线,\W非字类字符
  • 记一点看得懂就行,剩下查资料+AI

列表

List<String> arrayList=new ArrayList<>();一般用这个(99%),这个是动态数组,可变大小，高效随机访问和快速尾部插入，但中间插入和删除慢【注意new ArrayList<>()那个<>里就是可以不指定类型的,类型在前面的<>里定义,后面new里是可以不写的】
另一个是LinkedList<>();是双向链表，插入删除高效但访问慢,定义也是new LinkedList<>();
ArrayList:要添加就是.add(元素);访问元素.get(i),记得索引从0开始;修改.set(i,元素),把索引i的改成元素;删除.remove(i),把第i个删了;要元素数量.size();判断元素是否在表里.contains(元素);清空.clear();转化为数组.toArray();
LinkedList:.add(val)/.addLast(val)在尾部加;.addFirst(val)在头部加;.removeFirst()移除并返回头部;.removeLast()移除并返回尾部;.getFirst()获取头部;.getLast()获取尾部;要大小.size();访问第i个.get(i);.poll()和.remove()都是删除并返回头部;判断是否包含.contains(元素);清空.clear();

集合

Set<String> hashset=new HashSet<>();
存储不重复的元素，为无序集合，有高效查找和插入，但无序
HashSet是不允许有重复元素的集合，允许有null值，用法同ArrayList，只不过add重复元素则不会再添加同时add函数返回false,如果add成功添加返回true
HashSet的contains平均复杂度O(1),最坏O(logn),一般认为就是接近O(1);但是!ArrayList的contains就是O(n)遍历了

映射

Map<String,Integer> hashMap=new HashMap<>();
基于哈希表实现的键值对存储，高效查找、插入、删除，但无序;key不允许重复，value可以重复
初始化:Map<Integer,String> map=Map.of(1,"aaa",2,"bbb"),则map里有两个键值对,为 1 -> “aaa”,2 -> “bbb”;
添加键值对.put(key,value);获取key对应的value.get(key);删除key对应value.remove(key);.getOrDefault(Object key, V defaultValue),获取指定key对应的value,如果找不到key,则返回设置的默认值;.clear()和.size()仍然有效;返回所有value值.values();判是否有key.containsKey();
【注意put】若插入的key对应的value已存在,则执行value替换操作,返回旧的value值;如果不存在则插入并返回null

栈

Stack<Integer> stack=new Stack<>();为后进先出,.empty()是否为空,.peek()查顶端,.pop()弹出并返回顶端;.push(i)压入栈,.search(i)返回对象位置从1开始,.clear()清空

队列

Queue<String> queue=new LinkedList<>();先进先出,.offer(i)入队,.poll()删除并返回队头,.peek()查队头,.isEmpty()是否为空,.size()大小

一些力扣思考与tip

  • 若第一眼看上去两层循环模拟，但其中一个增必会导致另一个减小的(比如和为定值) -> 双指针,能将O(n2)降低为O(n)【一定要记得排序!】
  • 去重思路: HashSet,HashMap(也可以判断重复元素) ; 跳过相同元素
  • 注意new ArrayList<>()那个<>里就是可以不指定类型的,类型在前面的<>里定义,后面new里是可以不写的
  • 数组长度是a.length;字符串长度是s.length();注意带不带括号
  • 字符串子串必须连续,子序列不用连续(记忆:只要是串,那就必须穿起来,连续)
  • 字符串的字符数量统计支持++sCount[s.charAt(i) - 'a'];【注意这样写下标是0-25】
  • 字符串的.length()复杂度是O(1),放心用
  • int的极值Integer.MAX_VALUE,用于初始化;极小值是Integer.MIN_VALUE
  • 动态规划:连续区间内最值+只要结果,不要求得到那个连续区间是哪一个(无后效性)
  • Arrays.sort()自定义排序的话,sort里第一个变量还是要排的数组,第二个是实现Comparator并实现里面的public int compare(变量1,变量2)函数,返回>0的情况则认为排序时变量1>变量2,返回<0的情况则认为排序时变量1<变量2【具体记:要升序1减2,要降序2减1】
  • 螺旋矩阵:不是用vis[][]在那标记,而是弄4个边界变量,走完一条之后++或–边界变量 ; 不要额外开num变量计数了,用ArrayList的.size()计数 ; 每走完一条就要判一次退出(一个螺旋判4次)不是一个螺旋才判一次
  • 搜索二维矩阵:从(0,0)开始有问题,若要增大,不知道该往右还是往下 -> 从右上角(0,n-1)开始,要小就往左,要大就往下,超边界就false
  • 环形链表:Floyd判圈,快慢指针 -> 两个指针,快指针每次走2步,慢指针每次走1步,若快追上慢,则有环,否则到链表尾,无环(注意快指针走2步,要加一些非null判断) -> 找循环位置:设置前面的无环线为a,环中slow走的是b,环中剩下的是c,则slow走的长度是a+b,fast走的是a+b+c+b,slow*2=fast得到a=c,则再弄一个指针从head开始,和slow同步走,这俩最终会在循环处相遇
  • 删链表的倒数第N个结点:操作链表时，常用技巧是添加一个哑节点dummy node(node0)，它的next指向链表的头节点，这样就不用对头节点进行特殊判断了
  • 两两交换链表中的节点:也是用哑结点,每次取2个进行操作,注意结点交换想不明白就画图分析!
  • 合并K个链表:以合成2个链表为基础 -> 简单就前缀和那样合并,复杂度低点就二分那样合并
  • LRU缓存:HashMap+双向链表,HashMap通过key得到双向链表里的节点,双向链表存使用顺序,两个dummy节点head和tail,用的节点就挪到head后(挪->先删再加),新加节点也是head后,超容量就删tail前
  • 栈访问二叉树:root开始(注意while外面root不用入栈),若当前节点非null,入栈,left,否则出栈,right;条件是节点存在或栈非空 -> 入栈时访问是前序,出栈时访问是中序 -> 后序稍复杂,用递归吧
  • 验证二叉搜索树:二叉搜索树中序遍历必得到升序序列,可验证中序遍历为升序序列来判
  • 二叉搜索树中第K小的元素:中序遍历第k个(方法1)就够了,记录子树节点数(方法2)已经是会讲就差不多了,手写AVL树(平衡二叉搜索树)(方法3)面试中纯自杀【要会选复杂度合适的方案】
  • LCA最近公共祖先:1.递归:当前为空,p,q返回当前 -> 递归在左右里找pq -> 左右都有,返回当前 -> 只找到一个,返回非空那个 ; 2.先遍历一遍记父节点 -> p往上跳到root -> q往上跳,遇到访问过的第一个就返回那个
  • 二叉树中最大路径和:额外开一个递归弄当前node的最大贡献值:为null返回0,递归lr贡献值和0取最大,以当前node为拐点的最大路径和为node.val+l+r,和ans取最大,返回是node.val+max(l,r);主函数跑一遍递归返回ans即可
  • 岛屿数量:不是再开一个vis[][]判断了,而是搜到’1’直接变成’0’
  • 腐烂的橘子:多源广搜:把所有起点先全入队当第一层,然后正常广搜
  • 课程表:有向图DFS判环的vis一定是3种状态:未访问0,在当前路1,已完成2三种,在递归最后需要标记已完成(vis当前=2) ; 广搜是不断把入度为0的入队,扩展,最后判入过队数是否=n
  • 实现Trie树(字典树):相当于写26叉树,根据下一个字符是啥来指向下一个节点,然后每个节点包括一个boolean isEnd来标记是否是字符串的结尾;检查前缀和串存在都是一样,只不过检查串加一个判isEnd
  • 根据已有List新建一个复制:new ArrayList<>(mid),为了防止操作mid导致关联修改 ; 后面的mid(List)换成set也可以转换
  • 递归小tip:若要全排列,则递归里才有for循环;若没有全排列,递归里只有当前位置和下一位置的递归,没有for循环
  • new String(char[])可以把字符数组直接变成String
  • 取String子串的.substring(l,r)是左闭右开,从i取到j是.substring(i,j+1)
  • n皇后: 1.当前层只影响后续层,所以循环弄vis标记的时候只从当前层往后到n即可,不是从0开始 ; 2.回溯应只回溯当前层的影响,不能覆盖前序层,因此vis不是01而是要记录层数
  • 标准二分查找,找第1个>=的下标:while里有=,下面if没=,l和r都有+1-1 ; 注意l必有+1,l和r的=和r的-1才有问题 -> 如果是左闭右闭区间,l和r的=和r的-1都有;左闭右开则l和r的=和r的-1都没有 -> 就写==,>,<三种情况,若要找符合的第1个就是在==里面不break,而是往左走,加上r=mid-1继续循环;找符合的最后一个也是==里不break而是往右走,加上l=mid+1继续循环
  • 寻找两个正序数组的中位数:相当于找两个数组的第k小数,由于两个数组有序还要log复杂度,因此每次判定排除掉k/2个位置;将偶数和奇数的情况合并,k为(n+m+1)/2和(n+m+2)/2分别算一次后加起来*0.5;让len1的长度小于len2,这样就能保证如果有数组空了,一定是len1,此时返回nums2的第k个就得;k==1特判返回nums1[l1]和nums2[l2]中的较小那个;i和j的长度都是要+min(len,k/2)-1的,然后递归的k也是按位置减少的k-(i-l1+1)
  • 最小栈,O(1)寻找栈内最小值:弄两个栈,一个正常,另一个就弄最小值(初始有MAX_VALUE),就和正常栈顶判最小值,同步进出就行
  • Java字符串比较必须用A.equals(B)才行【基本类型用==(8个基本类型),对象内容用.equals()包括String,List,Map,包装类(如Integer,Long),自定义类】 ; 而且字符串的+是O(n)复杂度的,在循环里做字符串拼接,必须用可变字符串StringBuilder【记得最后加上.toString()!!】,不用StringBuffer是因为StringBuilder更快(因为不是线程安全的,不加锁,所以快;不过这俩语法是一样的) ; LinkedList需要addLast()然后reverse()的就直接变成addFirst()
  • O(n)求出数组各个元素右侧第一个更大的元素及其坐标:单调栈,存下标,是一个单调递减栈,然后遇到一个i处的若比栈顶大,则循环弹出栈顶,确定栈顶的ans,最后将i加入单调栈
  • 数据流的中位数:一个大根堆quemin放小于中位数的,一个小根堆quemax放大于中位数的 ; 每次要加num和quemin.peek()比较,num小就放quemin里,num大就放quemax里,然后平衡quemin和quemax(一个往另一个放),确保这俩的.peek()都和中位数有关 ; 初始0时先放quemin,然后中间维护也是奇数时quemin长度为quemax+1,此时要中位数就是quemin.peek(),偶数时要中位数就是(quemin.peek()+quemax.peek())*0.5
  • 【动态规划dp】相当于带备忘录的递归,就是:拆分子问题,记住过往,减少重复计算,记住求过的解是核心(就是先想递归,然后想一下递归是不是存在大量重复计算,然后用一个结构存下那个重复计算的结果,即变成dp)【即已经算出的结果就不会改变了,然后有状态转移方程f(i)=f(i-1)+f(i-2)这种依赖前面结果的】
  • 乘积最大子数组++思路: 对于最后的答案区间[l,r]，区间两端(不包含l和r): 1.不会任何一边有正数，有的话肯定能乘正数变大或不变 ; 2.不会同时两边有负数，有的话肯定能乘俩负数变大或不变 -> 所以答案的一端一定是0或者整个数组起止点，因此从左右两端算乘积，遇到0就重新当起点，ans实时更新即可 -> ans=Math.max(Math.max(ans,nums[i]),Math.max(l,r));
  • 分割等和子集:经典NP完全问题,类似01背包,数学上不存在多项式算法(就是时间复杂度只带n),只能用非多项式时间复杂度的算法(就是时间复杂度里带k),如动态规划
  • 最长有效括号:从左往右,记l为’(‘数,r为’)'数,l>r就继续,l==r这时必定合法,更新ans,一旦r>l,那说明当前右括号多了,必不合法,重置l=r=0 ; 然后为了避免l一直>r更不了ans,从左往右和从右往左都弄一遍,即可
  • 最长回文子串:就枚举中心,往l和r两边扩展,扩完弄ans即可,O(n2) ; 马拉车虽然是O(n)但太复杂,面试不考【进一步说明题解并不是全得会】
  • 最长公共子序列:考研题,两重循环,字符相等就是左上+1,不等就是左和上取最大(就是考研那个箭头图)
  • 编辑距离:f[i][j]表示A的前i个字符和B的前j个字符的编辑距离;定义f[n+1][m+1];边界f[i][0]=i,f[0][j]=j;循环若i和j字符相等,f[i][j]=f[i-1][j-1],若字符不等则f[i][j]=min(f[i-1][j]+1,f[i][j-1]+1,f[i-1][j-1]+1);返回f[n][m]
  • 只出现一次的数字:看标签有位运算:异或^,注意a^0=a

  • 多数元素:投票法:不同的两个数同归于尽,因为众数最多且比其他加起来都多,所以最后剩下的一定是众数
  • 下一个排列:先倒着找第一个a[i] < a[i+1]的(如果找不到,那就翻转整个数组) -> 然后从a[i+1]到a[n]倒着找第一个a[j]>a[i]的,交换a[i]和a[j]【i>=0才做】 -> 然后将a[i+1]到a[n]翻转
  • 寻找重复数:O(n)时间+O(1)空间:连边i->nums[i],即转化成环形链表找环的入口,就是2*(a+b)=a+b+c+b然后得到a=c那个,先都从0开始快慢走到相遇,然后slow变0再同速走到相遇,然后就是slow
