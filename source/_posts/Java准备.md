## Java正则特殊
在 Java 中正则表达式中则需要有两个反斜杠才能被解析为其他语言中的转义作用，在 Java 的正则表达式中，两个(反斜杠)代表其他语言中的一个(反斜杠)，表示一位数字的正则表达式是(两个反斜杠)d，而表示一个普通的反斜杠是(两个反斜杠)
**一些正则语法**:
- `^`匹配开始,`$`匹配结尾
- 跟在要匹配的字符后面:`*`匹配0~多次,`+`匹配1~多次,`?`匹配0~1次,`{n}`正好匹配n次,`{n,}`匹配n~多次,`{n,m}`匹配n~m次,`.`匹配单个任意字符
- `x|y`匹配x或y(`z|food`匹配z或food,而`(z|f)ood`匹配zood或food)
- `[xyz]`匹配包含的一个字符,`[^xyz]`匹配不包含的一个字符,`[a-z]`匹配指定范围内一个,`[^a-z]`匹配不在范围内的一个
- `\d`数字字符0-9,`\D`非数字(不是0-9),`\n`换行,`\s`空白字符,`\S`非空白字符,`\w`字类字符大小写字母+数字+下划线,`\W`非字类字符
- 记一点看得懂就行,剩下查资料+AI

总体流程:正则式String -> `Pattern.compile(正则式)` -> `p.matcher(要匹的串)` -> `m.方法`进行操作

## 数据结构(涉及去重一定要想HashSet和HashMap)

### 数组
`int[] array=new int[5];`Java数组也是要指定大小的,只不过是在后面赋值指定
新建定值数组`new int[]{1,2,3}`
数组new时的默认初值:整数类型 -> `0`;浮点型 -> `0.0`;char -> `'\u0000'`;boolean -> `false`;对象类型(如String) -> 全null

### 列表
`List<String> arrayList=new ArrayList<>();`一般用这个(99%),这个是动态数组,可变大小，高效随机访问和快速尾部插入，但中间插入和删除慢【注意`new ArrayList<>()`那个<>里就是可以不指定类型的,类型在前面的<>里定义,后面new里是可以不写的】
另一个是`LinkedList<>()`;是双向链表，插入删除高效但访问慢,定义也是`new LinkedList<>();`
ArrayList:要添加就是`.add(元素)`;访问元素`.get(i)`,记得索引从0开始;修改`.set(i,元素)`,把索引i的改成元素;删除`.remove(i)`,把第i个删了;要元素数量`.size()`;判断元素是否在表里`.contains(元素)`;清空`.clear()`;转化为数组`.toArray()`;
LinkedList:`.add(val)/.addLast(val)`在尾部加;`.addFirst(val)`在头部加;`.removeFirst()`移除并返回头部;`.removeLast()`移除并返回尾部;`.getFirst()`获取头部;`.getLast()`获取尾部;要大小`.size()`;访问第i个`.get(i)`;`.poll()`和`.remove()`都是删除并返回头部;判断是否包含`.contains(元素)`;清空`.clear()`;

### 集合
`Set<String> hashset=new HashSet<>();`
存储不重复的元素，为无序集合，有高效查找和插入，但无序
HashSet是不允许有重复元素的集合，允许有null值，用法同ArrayList，只不过add重复元素则不会再添加同时add函数返回false,如果add成功添加返回true
HashSet的contains平均复杂度O(1),最坏O(logn),一般认为就是接近O(1);但是!ArrayList的contains就是O(n)遍历了

### 映射
`Map<String,Integer> hashMap=new HashMap<>();`
基于哈希表实现的键值对存储，高效查找、插入、删除，但无序;key不允许重复，value可以重复
初始化:`Map<Integer,String> map=Map.of(1,"aaa",2,"bbb")`,则map里有两个键值对,为 1 -> "aaa",2 -> "bbb";
添加键值对`.put(key,value)`;获取key对应的value`.get(key)`;删除key对应value`.remove(key)`;`.getOrDefault(Object key, V defaultValue)`,获取指定key对应的value,如果找不到key,则返回设置的默认值;`.clear()`和`.size()`仍然有效;返回所有value值`.values()`;判是否有key`.containsKey()`;
【注意put】若插入的key对应的value已存在,则执行value替换操作,返回旧的value值;如果不存在则插入并返回null

### 栈
`Stack<Integer> stack=new Stack<>();`为后进先出,`.empty()`是否为空,`.peek()`查顶端,`.pop()`弹出并返回顶端;`.push(i)`压入栈,`.search(i)`返回对象位置从1开始,`.clear()`清空

### 队列
`Queue<String> queue=new LinkedList<>();`先进先出,`.offer(i)`入队,`.poll()`删除并返回队头,`.peek()`查队头,`.isEmpty()`是否为空,`.size()`大小

#### 优先队列PriorityQueue，也是堆
检索绝对是按顺序(默认升序,如果是自定义排序的话,`public int compare(变量1,变量2)`函数,返回>0认为变量2优先,返回<0认为变量1优先【具体记:要升序1减2,要降序2减1】)
增`.offer(i)`;访问队头(极值)`.peek()`;返回并删除队头`.poll()`;`.remove()`,`.add(i)`,`.contains(i)`,`.toArray()`都可用
> 若自定义比较，需要实现`Comparator<>()`接口

## iterator，Object类，NIO Files类，泛型，序列化，Java网络编程(TCP,UDP,Socket)

UDP是不可靠、无连接、快,可能丢包、乱序但延迟低,常用于视频、语音、游戏
Socket就是应用层和传输层的接口,用来进行网络通信;Socket编程一般分客户端和服务端

## Java多线程:是为了充分利用CPU
进程是程序的一次执行，为资源分配单位
线程是进程中的执行单元，为CPU调度单位
-> 一个进程可包含一个或多个线程
### ?如何创建线程
1. 继承Thread类
2. 实现`Runnable`接口,这个更推荐,因为Java是单继承,实现Runnable更灵活
### 线程生命周期
线程先进行创建,后调用`start()`即就绪,再执行`run()`即开始运行,期间可能阻塞也可能继续`run()`,运行完后`(run()执行完)`即终止
### 核心问题:线程安全
-> 多个线程同时访问共享资源，则结果不可预期 -> 多线程下要保护共享资源
使用`synchronized`修饰即可保证同一时间只有一个线程访问共享代码
**小tip**:`.sleep()`不会释放锁;`.wait()`会释放锁 -> 这里的锁就是对共享资源的锁

## 一些力扣思考与tip
- 若第一眼看上去两层循环模拟，但其中一个增必会导致另一个减小的(比如和为定值) -> 双指针,能将O(n2)降低为O(n)【一定要记得排序!】
- 去重思路: HashSet,HashMap(也可以判断重复元素) ; 跳过相同元素
- 注意`new ArrayList<>()`那个<>里就是可以不指定类型的,类型在前面的<>里定义,后面new里是可以不写的
- 数组长度是`a.length`;字符串长度是`s.length()`;注意带不带括号
- 字符串子串必须连续,子序列不用连续(记忆:只要是串,那就必须穿起来,连续)
- 字符串的字符数量统计支持`++sCount[s.charAt(i) - 'a'];`【注意这样写下标是0-25】
- 字符串的`.length()`复杂度是O(1),放心用
- int的极值`Integer.MAX_VALUE`,用于初始化;极小值是`Integer.MIN_VALUE`
- 动态规划:连续区间内最值+只要结果,不要求得到那个连续区间是哪一个(**无后效性**)
- `Arrays.sort()`自定义排序的话,sort里第一个变量还是要排的数组,第二个是实现Comparator并实现里面的`public int compare(变量1,变量2)`函数,返回>0的情况则认为排序时变量1>变量2,返回<0的情况则认为排序时变量1<变量2【具体记:要升序1减2,要降序2减1】
- 螺旋矩阵:不是用`vis[][]`在那标记,而是弄4个**边界变量**,走完一条之后++或--边界变量 ; 不要额外开num变量计数了,用ArrayList的`.size()`计数 ; 每走完一条就要判一次退出(一个螺旋判4次)不是一个螺旋才判一次
- 搜索二维矩阵:从(0,0)开始有问题,若要增大,不知道该往右还是往下 -> 从右上角(0,n-1)开始,要小就往左,要大就往下,超边界就false
- 环形链表:Floyd判圈,快慢指针 -> 两个指针,快指针每次走2步,慢指针每次走1步,若快追上慢,则有环,否则到链表尾,无环(注意快指针走2步,要加一些非null判断) -> 找循环位置:设置前面的无环线为a,环中slow走的是b,环中剩下的是c,则slow走的长度是a+b,fast走的是a+b+c+b,slow*2=fast得到a=c,则再弄一个指针从head开始,和slow同步走,这俩最终会在循环处相遇
- 删链表的倒数第N个结点:操作链表时，常用技巧是添加一个哑节点dummy node(node0)，它的next指向链表的头节点，这样就不用对头节点进行特殊判断了
- 两两交换链表中的节点:也是用哑结点,每次取2个进行操作,注意结点交换想不明白就画图分析!
- 合并K个链表:以合成2个链表为基础 -> 简单就前缀和那样合并,复杂度低点就二分那样合并
- LRU缓存:HashMap+双向链表,HashMap通过key得到双向链表里的节点,双向链表存使用顺序,两个dummy节点head和tail,用的节点就挪到head后(挪->先删再加),新加节点也是head后,超容量就删tail前
- 栈访问二叉树:root开始(注意while外面root不用入栈),若当前节点非null,入栈,left,否则出栈,right;条件是节点存在或栈非空 -> 入栈时访问是前序,出栈时访问是中序 -> 后序稍复杂,用递归吧
- 验证二叉搜索树:二叉搜索树中序遍历必得到升序序列,可验证中序遍历为升序序列来判
- **二叉搜索树中第K小的元素**:中序遍历第k个(方法1)就够了,记录子树节点数(方法2)已经是会讲就差不多了,手写AVL树(平衡二叉搜索树)(方法3)面试中纯自杀【要会选复杂度合适的方案】
- LCA最近公共祖先:1.递归:当前为空,p,q返回当前 -> 递归在左右里找pq -> 左右都有,返回当前 -> 只找到一个,返回非空那个 ; 2.先遍历一遍记父节点 -> p往上跳到root -> q往上跳,遇到访问过的第一个就返回那个
- 二叉树中最大路径和:额外开一个递归弄当前node的最大贡献值:为null返回0,递归lr贡献值和0取最大,以当前node为拐点的最大路径和为`node.val+l+r`,和ans取最大,返回是`node.val+max(l,r)`;主函数跑一遍递归返回ans即可
- 岛屿数量:不是再开一个vis[][]判断了,而是搜到'1'直接变成'0'
- 腐烂的橘子:多源广搜:把所有起点先全入队当第一层,然后正常广搜
- 课程表:有向图DFS判环的vis一定是3种状态:未访问0,在当前路1,已完成2三种,在递归最后需要标记已完成(vis当前=2) ; 广搜是不断把入度为0的入队,扩展,最后判入过队数是否=n
- 实现Trie树(字典树):相当于写26叉树,根据下一个字符是啥来指向下一个节点,然后每个节点包括一个`boolean isEnd`来标记是否是字符串的结尾;检查前缀和串存在都是一样,只不过检查串加一个判isEnd
- 根据已有List新建一个复制:`new ArrayList<>(mid)`,为了防止操作mid导致关联修改 ; 后面的`mid(List)`换成set也可以转换
- 递归小tip:若要全排列,则递归里才有for循环;若没有全排列,递归里只有当前位置和下一位置的递归,没有for循环
- `new String(char[])`可以把字符数组直接变成String
- 取String子串的`.substring(l,r)`是左闭右开,从i取到j是`.substring(i,j+1)`
- **n皇后**: 1.当前层只影响后续层,所以循环弄vis标记的时候只从当前层往后到n即可,不是从0开始 ; 2.回溯应只回溯当前层的影响,不能覆盖前序层,因此vis不是01而是要记录层数
- 标准二分查找,找第1个>=的下标:while里有=,下面if没=,l和r都有+1-1 ; 注意l必有+1,l和r的=和r的-1才有问题 -> 如果是**左闭右闭**区间,l和r的=和r的-1都有;**左闭右开**则l和r的=和r的-1都没有 -> 就写==,>,<三种情况,若要找符合的第1个就是在==里面不break,而是往左走,加上r=mid-1继续循环;找符合的最后一个也是==里不break而是往右走,加上l=mid+1继续循环
